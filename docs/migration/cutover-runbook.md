# Bigbang 切流与回滚 Runbook

## 切流前检查
- 新平台健康检查通过（web/orchestrator/workers/postgres/redis/minio）。
- 核心接口冒烟通过：创建任务、拉取状态、消费事件流、任务取消。
- 样本任务验证：AI 队列至少 20 次无崩溃。

## 数据迁移策略
- 迁移源系统进入只读窗口（冻结写入）。
- 迁移基础数据：
  - 用户元数据（users）
  - 项目索引（projects）
  - 历史产物映射（artifacts 索引，不强制迁移文件二进制）
- 迁移后校验记录总量与抽样一致性。

## 切流步骤
1. Cloudflare 将主域灰度 10% 到新入口。
2. 观察 30-60 分钟（错误率、队列堆积、SSE 中断率）。
3. 扩大到 50%，再到 100%。
4. 将备用入口标记为降级路径。

## 回滚策略
- 任一指标触发阈值立即回切：
  - API 5xx > 2%
  - 任务失败率 > 8%
  - 队列等待 > 5 分钟
- 回滚后保留事件日志用于根因分析。
